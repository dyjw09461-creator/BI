> 积兑分流程及说明：
>
> 概述：由于积分平台存在有多个商家的积分数据，为体现公平性，积分平台采用先进先出的策略，即先积哪个商家的积分，兑换礼品的时候就优先使用该商家的积分兑换。为方便今后扩展，我们引入账户的概念，即积分平台会员的积分，全部采用账户的方式记录。平台会员和平台商家在积分平台上都拥有自己的积分账户。可以在平台上查看对应的积分与兑分日志。需要注意如下几点：
>
> 为简化核算，方便对账，我们引入财务借贷的概念，有借必有贷，借贷必相等的原则。即会员购买商品积分，相当于商家积负分(等于增加商家的负债)。会员兑换礼品花费积分，相当于商家负分减少(等于减少商家负债)。

1.  会员每购买商品积分一次，相对应有一个商家积负分，总积分为0。

2.  会员每兑换一次积分，相对应商家有一个积分消费，总积分为0。

3.  会员每次增加积分，会增加有效积分余额与历史积分。

4.  会员每次兑分，有效积分余额立即减少对应分值(以避免超兑)，此部分积分进入冻结状态；当管理员审批订单后，进入在途状态；当订单配送完成时，这时才真正从会员积分余额里扣减。

> 举例说明：
>
> A会员有1500积分余额，A会员在2017.9.1兑换600积分，在2017.9.2订单审核通过，进入配送状态，在2017.9.5日完成订单配送。
>
> 复杂版表格如下：

|      |           |          |          |          |     |
|------|-----------|----------|----------|----------|-----|
| 会员 | 时间      | 积分余额 | 冻结积分 | 在途积分 |     |
| A    | 2017.8.31 | 1500     | 0        | 0        |     |
| A    | 2017.9.1  | 900      | 600      | 0        |     |
| A    | 2017.9.2  | 900      | 0        | 600      |     |
| A    | 2017.9.5  | 900      | 0        | 0        |     |

|      |           |          |          |          |     |
|------|-----------|----------|----------|----------|-----|
| 商家 | 时间      | 积分余额 | 冻结积分 | 在途积分 |     |
| B    | 2017.8.31 | -1500    | 0        | 0        |     |
| B    | 2017.9.1  | -900     | -600     | 0        |     |
| B    | 2017.9.2  | -900     | 0        | -600     |     |
| B    | 2017.9.5  | -900     | 0        | 0        |     |

4.. 会员每次兑分，有效积分余额立即减少对应分值(以避免超兑)，商家兑分立即增加。
